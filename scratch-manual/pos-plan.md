## ภาพรวมคอนเซ็ปต์
- login ที่ `index.html` เดิม → **จอเลือกทาง**: 🛒 ขาย POS / 🏪 โปรแกรมจัดการร้าน
- ใช้บัญชี/พนักงาน/สิทธิ์/ฐานข้อมูล + Firebase ชุดเดียวกับแอปเดิม
- **POS เน้นใช้บนคอมพิวเตอร์** (จอกว้าง · คีย์บอร์ด · เครื่องยิงบาร์โค้ด) — ต่างจากโปรแกรมจัดการร้านที่เน้นมือถือ
- POS **แยกตาม 3 สาขา** (0=ทรัพย์พัฒนา, 1=บางปู, 2=อินดี้) — เลือกสาขาก่อนเริ่มขาย

## โครงจอ (คุย/ทำ mockup แล้ว)
1. ✅ mockup — จอเลือกทาง (POS / จัดการร้าน)
2. ✅ mockup — จอเลือกสาขาก่อนขาย (จำสาขาล่าสุดของเครื่อง · โชว์สถานะเปิดร้าน/ยอดวันนี้)
3. ✅ mockup — จอขาย POS บนคอม (ซ้าย=ค้นหา/ยิงบาร์โค้ด+สินค้า · ขวา=ตะกร้า+ยอด+ปุ่มชำระ · ปุ่มลัดคีย์บอร์ด)

## โครงจอ (ยังไม่ทำ mockup)
- ⬜ จอคลังสินค้า (สต็อกแยกสาขา + ล็อต/FIFO + วันหมดอายุ)
- ⬜ จอรับยาเข้า (จากซื้อ / จากปรับสต็อก)
- ⬜ จอนับสต็อกบนมือถือ (สแกน + กรอกจำนวน)
- ⬜ จอขอแบ่งยาข้ามสาขา
- ⬜ จอข้อมูลสินค้า (ป้าย ข.ย.9/11 · อาหารเสริม · บาร์โค้ด · ราคา/ทุน · หน่วยขาย)
- ⬜ จอออกใบกำกับภาษี (ที่อยู่สาขา vs ที่อยู่นิติบุคคล)
- ⬜ จอลูกค้าสมาชิก (โปรไฟล์ · แต้ม · ประวัติซื้อ · แพ้ยา · โรคประจำตัว)
- ⬜ แถบผูกลูกค้าบนจอขาย POS (ค้นเบอร์/ชื่อ/สแกน QR + เตือนแพ้ยา)
- ✅ mockup — จอรายการสินค้า (ตาราง + ค้นหา + นำเข้า Excel + สร้าง/แก้ไข/ลบ)
- ✅ mockup — จอสร้าง/แก้ไขสินค้า (ล้อ hygeia + สต็อกแยกสาขา + ธง ข.ย.)
- ✅ mockup — ระบบรายงาน: ศูนย์รายงาน + บัญชี ข.ย. (GPP) + บันทึกอุณหภูมิ + สรุปยอดขาย/กำไร

### ระบบรายงาน — เคาะโครงแล้ว 17 ก.ค. (แยก 2 กลุ่ม)
- **กลุ่ม GPP (บังคับตามกฎหมาย)** — จุดขายหลักของระบบ เพราะเดี๋ยวนี้ทำมือ:
  - บัญชี **ข.ย.9** (ซื้อยา — จากรับเข้า) · **ข.ย.10/11** (ขายควบคุมพิเศษ/อันตราย — **ออกอัตโนมัติจากบิลขายที่จับยาติดธง ข.ย.**) · **ข.ย.12** (ตามใบสั่งแพทย์ + ฟอร์มเปล่า) · **ข.ย.13** (ขายส่ง)
  - ทุกช่องมาจากข้อมูลที่มีอยู่: วันที่/ยา/จำนวน(บิล) · ตัวยา/ทะเบียน(ข้อมูลยาในสินค้า) · ผู้จ่าย(เภสัชที่ล็อกอิน) · ผู้ซื้อ/อาการ(ช่องบันทึกอาการตอนขาย)
  - พิมพ์ตรงฟอร์มราชการ / PDF / Excel · เลือกเดือน+สาขา
  - **บันทึกอุณหภูมิ & ความชื้น** — วันละ 3 ครั้ง (ภายในร้าน<30°C · เก็บยาสำรอง<30°C · ตู้เย็น 2-8°C · ชื้น<60%) · เช็กเกณฑ์อัตโนมัติ(เขียว/แดง) · ปฏิทินเดือน + เตือนถ้าลืม(ใช้ push เดิม) · ทำบนมือถือ · แยกสาขา · พิมพ์รายเดือน
- **กลุ่มยอดขาย/กำไร/สินค้า/ลูกค้า** (ต่อยอดแดชบอร์ดเดิม Chart.js):
  - ยอดขายรายวัน/เดือน/ปี+เทียบปี · กำไร-ต้นทุน(จากทุน FIFO) · แยกผู้ขาย · รายชั่วโมง · จำนวนบิล/เฉลี่ย · รับเงิน-ทอน
  - สินค้าขายดี/กำไรดี/ใกล้หมด(ต่ำกว่าจุดสั่งซื้อ)/ใกล้หมดอายุ(ล็อต)/ไม่เคลื่อนไหว/มูลค่าสต็อก
  - ลูกค้า: สรุปการซื้อ · รายชื่อ+เบอร์+วันซื้อ(Excel) · แต้มสะสม
  - **ทุกอันเลือกรายสาขา หรือ รวม 3 สาขา**
- **แบบฟอร์มพิมพ์ GPP — ทำตรงฟอร์มจริงที่เจ้าของส่ง PDF มาแล้ว** (แนวนอน A4):
  - **ข.ย.๙ (ซื้อยา)**: ลำดับ·วันที่ซื้อ·ชื่อผู้ขาย·ชื่อยา·**เลขครั้งที่ผลิต(ล็อต)**·จำนวน·ลายมือชื่อเภสัช·หมายเหตุ → เติมจากรับเข้าซื้อ (เลขล็อตมาจากระบบ FIFO)
  - **ข.ย.๑๑ (ขายยาอันตราย)**: **1 แผ่นต่อยา 1 ล็อต** — หัวกระดาษ(ชื่อยา/ผู้ผลิต/ล็อต/ขนาดบรรจุ/ได้มาจาก/จำนวนรับ/วันที่รับ) + ตาราง(วันที่ขาย·จำนวนขาย·ชื่อสกุลผู้ซื้อ·ลายเซ็น·หมายเหตุ) → เข้ากับระบบล็อต FIFO เป๊ะ · ผู้ซื้อได้จากสมาชิก/กรอกตอนขาย
  - **ข.ย.๑๒ (ตามใบสั่งแพทย์)**: หัวคอลัมน์ซ้อน **ผู้สั่งยา**(ชื่อ+ที่อยู่) · **ผู้ใช้ยา**(ชื่อ+อายุ+ที่อยู่) · ชื่อยา+จำนวน · ผู้ใช้ที่เป็นสมาชิกเติมอัตโนมัติ
  - **บันทึกอุณหภูมิ**: วันที่ 1-31 = คอลัมน์ · 3 ครั้ง/วัน = แถว · 5 บล็อก (ภายในร้าน/สำรอง อุณหภูมิ+ความชื้น + ตู้เย็น 2-8°C) · เกินเกณฑ์ขึ้นแดง · **ระบบออก "กราฟรายเดือน" ด้วย** (ค่าสูงสุดของวัน 3 จุด) ตามที่เจ้าของส่งมา
  - **ข.ย.๑๐ (ควบคุมพิเศษ)**: โครงเดียวกับ ข.ย.๑๑ เป๊ะ (1 แผ่น/ยา/ล็อต) ต่างแค่ธงประเภท → ทำครั้งเดียวใช้ได้ทั้งคู่ ระบบแยกตามธง
  - **ข.ย.๑๓ (ขายส่ง)**: หัวมีที่อยู่ร้านเต็ม (docSettings) + เลขทะเบียนตำรับยา · หัวกระดาษต่อยา/ล็อต · ตารางคอลัมน์ซ้อน **จ่ายไปให้**(ชื่อ+ที่อยู่/ประเภท) + **จำนวน**(ขาย/**คงเหลือสะสม**=คำนวณจากล็อต) · 2 ลายเซ็น (ผู้รับอนุญาต+เภสัช)
  - **ครบทั้ง 6 ฟอร์มแล้ว** ทำตรงฟอร์มจริง PDF ทุกใบ

### รายละเอียดระบบสมาชิกลูกค้า (ฟีเจอร์ #8) — เคาะแล้ว 17 ก.ค.
- **สมัครที่ไหนก็ถึงกันหมด** — ลูกค้า 1 คน = 1 โปรไฟล์กลาง (Firebase ชุดเดียว) · สมัครที่บางปู ไปซื้อที่อินดี้ก็เจอโปรไฟล์เดิม แต้มเดิม ประวัติเดิม
- **สมัคร: พนักงานกรอกที่ POS** — บนจอขายมีปุ่ม "สมัครสมาชิก" · กรอกชื่อ + เบอร์โทร (ไม่มีบัตร/QR)
- **ค้นลูกค้าด้วยเบอร์โทร (หลัก) หรือชื่อ** — ผูกเข้าบิลตอนขาย · ไม่ผูกก็ขายได้ (ลูกค้าทั่วไป)
- **สะสมแต้ม — อัตราตั้งเองได้** — แอดมินพิมพ์อัตราใน `config` เช่น "25 บาท = 1 แต้ม" ปรับได้ทุกเมื่อ (มีช่องให้ใส่) · ใช้แต้มแลกส่วนลด/ของ · เห็นแต้มคงเหลือบนจอขาย
- **ประวัติการซื้อ** — ซื้ออะไร วันไหน สาขาไหน กี่บาท — ดูจากโปรไฟล์ลูกค้า (ช่วยแนะนำ/ซื้อซ้ำ)
- **ข้อมูลสุขภาพ (สำคัญ/ความปลอดภัย)**:
  - **แพ้ยา** — บันทึกตัวยา/กลุ่มที่แพ้ · ตอนขายถ้าหยิบยาที่ตรงกับที่แพ้ → **เด้งเตือนแดง** กันจ่ายผิด
  - **โรคประจำตัว** — เบาหวาน/ความดัน/ไต ฯลฯ · เตือนยาที่ต้องระวัง (เฟสหลัง)
  - เป็น **ข้อมูลอ่อนไหว** → เก็บแยก + คุมสิทธิ์ด้วย Firestore rules (เหมือน memberSecure ของพนักงาน)

### 🔗 ระบบที่ต้องลิงก์ POS ↔ โปรแกรมจัดการร้าน (app.html) — เจ้าของย้ำ
> POS ขายอย่างเดียว แต่ "การจัดการ" อยู่ฝั่งโปรแกรมจัดการร้านเดิม — ต้องเชื่อมกัน ไม่แยกฐาน
- **สมาชิก**: POS = สมัคร/ค้น/ผูกบิล/สะสม-ใช้แต้ม · จัดการร้าน = จัดการรายชื่อสมาชิก, ตั้งอัตราแต้ม, แก้แพ้ยา/โรค, ดูประวัติ/รายงานลูกค้า
- **ยอดขาย/เงินสด**: บิล POS วิ่งเข้า `sales` + ลิ้นชักเงินสดของสาขาเดิมอัตโนมัติ (ระบบเงินเดิมไม่ต้องแก้)
- **สต็อก**: รับเข้า/ปรับ/นับ/แบ่งยา — ทำจากฝั่งจัดการร้าน · POS แค่ตัดสต็อกตอนขาย
- **สินค้า/ราคา**: master `products` ชุดเดียว · แก้ราคา/ป้าย ข.ย. จากจัดการร้าน · POS อ่านไปใช้
- **พนักงาน/สิทธิ์/สาขา**: ใช้ `members`/role/สาขา ชุดเดียวกับแอปเดิมทั้งหมด

### ระบบสินค้า (สร้าง/แก้ไข/ลบ) — เคาะโครงแล้ว 17 ก.ค. (ล้อ hygeia)
- **จอรายการ**: ตาราง (รหัส/ชื่อ/ประเภท/ทุน/ขาย/คงเหลือรวม 3 สาขา/สถานะ/จัดการ) · ค้นหา+ฟิลเตอร์ · ปุ่มสร้างใหม่(F1) · นำเข้า Excel
- **จอฟอร์ม** แบ่งกล่อง: ข้อมูลหลัก(ชื่อ/ชื่อพิมพ์/ประเภท/ชื่อสามัญ/รหัส) · บาร์โค้ด 1-3 · ราคา&หน่วย(ทุน/ขายปลีกรวม VAT/กำไรคำนวณให้/VAT toggle) · ธง ข.ย.9/11/13 + อาหารเสริม · **สต็อกแยก 3 สาขา + จุดสั่งซื้อ** · สถานะใช้งาน/ระงับ
- **จุดต่างจาก hygeia**: คงเหลือ**แยก 3 สาขา** (เดิมช่องเดียว) · คงเหลือแก้ตรงๆไม่ได้ ต้องผ่านรับเข้า/นับ/แบ่งยา (ให้ FIFO มีประวัติครบ) · ฟอร์มตั้งได้แค่จุดสั่งซื้อ
- **ลบ vs ระงับ**: มีประวัติขาย/สต็อก → แนะนำ "ระงับ" (ซ่อนจาก POS แต่เก็บประวัติ) · ลบถาวรเฉพาะที่ไม่เคยขาย+สต็อก 0
- **ตัดออกชั่วคราว** (ร้านยังไม่ใช้): ราคาส่ง · หน่วยอื่น · ชั้นวาง · TMT ID — เพิ่มทีหลังได้ถ้าต้องการ
- **แท็บ ข้อมูลยา** (เปิดเมื่อ "เป็นยา"): ข้อบ่งใช้ · ประเภทตามกฎหมาย · เลขทะเบียนยา · ธงรายงาน ข.ย.9/11/13 · คำเตือน/ผลข้างเคียง (ดึงคลัง drugWarnings) · **ชื่อสามัญ+ความแรง+หน่วย** (หลายตัว สูตรผสม)
- **แท็บ ฉลากยา** = **เชื่อมระบบฉลากเดิมที่แอปมีอยู่แล้ว** (ตัวประกอบวิธีใช้/dose · labelMaster · drugWarnings · เทมเพลต) — ตั้งวิธีใช้ครั้งเดียว (ทานครั้งละ/วันละ/ช่วงเวลา/มื้อ/คำแนะนำ/หมดอายุนับจากวันขาย) → พิมพ์ตอนขายที่ POS · ที่ POS ดึงชื่อ+ที่อยู่ลูกค้าเข้าฉลากอัตโนมัติ · รองรับหลายภาษา (ต่อยอดฉลากต่างชาติ)

## แผนข้อมูล (ร่าง — Firestore collections ใหม่)
> ยังไม่ล็อก · ปรับได้ตอนเคาะ
- `products` — สินค้า master ใช้ร่วมทุกสาขา (ชื่อ, บาร์โค้ด, ประเภท, ธง ข.ย., หน่วยขาย, ราคาปลีก, ราคาส่ง, ชื่อสามัญ)
- `stockLots` — ล็อตต่อสาขา (หัวใจ FIFO): productId, branch, qty, cost, expiry, receivedAt, ref
- `stockCounts` — การนับสต็อก (สาขา, วันที่, รายการนับ, ผลต่าง)
- `transfers` — ขอแบ่งยาข้ามสาขา (จากสาขา, ไปสาขา, รายการ, สถานะ)
- `purchases` — รับเข้าจากการซื้อ (ผู้ขาย, บิล, รายการ, เครดิต)
- `posSales` / ต่อยอด `sales` เดิม — บิลขาย POS (เข้าระบบยอดขาย/เงินสดเดิมด้วย)
- `branchInfo` (หรือ `config/branches`) — ที่อยู่รายสาขา
- `tempLogs` — บันทึกอุณหภูมิ/ความชื้น (สาขา, วันที่, 3 ครั้ง, จุดวัด, ค่า, เกิน/ผ่าน)
- รายงาน ข.ย. = query จาก `posSales`/`purchases` + ธง ข.ย. ใน`products` (ไม่ต้องมี collection ใหม่)
- `customers` — โปรไฟล์ลูกค้าสมาชิก (มีอยู่แล้ว): ชื่อ, เบอร์, แต้ม, สาขาที่สมัคร
- `customerSecure` — ข้อมูลสุขภาพอ่อนไหว (แพ้ยา, โรคประจำตัว) แยกเก็บ + คุมสิทธิ์
- `pointsLedger` (หรือฝังใน bill) — บันทึกได้/ใช้แต้มแต่ละครั้ง (โปร่งใส ตรวจย้อนได้)
- ประวัติการซื้อ = query จาก `posSales`/`sales` ด้วย customerId

## 🔀 Flow ระบบ (18 ก.ค. — เริ่มคุย flow, พักฟีเจอร์ไว้ก่อน)
- **แผนที่ระบบ**: login(index.html) → จอเลือกทาง → [POS: เลือกสาขา→จอขาย] / [จัดการร้าน: เวลา&วันลา · เงิน&ยอดขาย · คลังสินค้า · ลูกค้าสมาชิก · รายงาน&GPP · เอกสาร&ฉลากยา] · ฐาน Firebase เดียว
- **Flow ขาย POS** (6 ขั้น → fan-out 5): เลือกสาขา → ค้นหา/ยิงบาร์โค้ด → เพิ่มตะกร้า/ชุดสินค้า → ผูกสมาชิก(เตือนแพ้ยา) → กรอกที่ ข.ย.ต้องมี(เฉพาะยาควบคุม) → ชำระเงิน · กดจบบิลครั้งเดียว → ตัดสต็อก FIFO + เงินเข้าลิ้นชัก + ลง ข.ย. + พิมพ์ใบเสร็จ/ฉลาก + สะสมแต้ม
- **Flow ที่ยัง map ต่อ**: รับของเข้า · เปิด-ปิดร้าน · คืนสินค้า · นับสต็อก+แบ่งยาข้ามสาขา · ออกรายงาน GPP รายเดือน

### ตอนจบบิล — ฉลาก/ใบเสร็จ/ข.ย. (เคาะแล้ว 18 ก.ค.)
- **ข.ย. (①)**: มียาควบคุม/อันตรายในบิล → เช็กชื่อผู้ซื้อ+อาการ (สมาชิกได้อัตโนมัติ / ยังไม่กรอกเด้งให้กรอก) → ลง ข.ย.10/11 แยกตามล็อต + ชื่อเภสัช
- **ฉลากยา (②)**: **พนักงานเลือกพิมพ์ทีละตัว** (ไม่บังคับทุกตัว) · **★ 1 ยามีได้หลายฉลากตามอาการ** (เช่น Amoxicillin: เจ็บคอ/ฝีหนอง/ทางเดินปัสสาวะ) — เก็บเทมเพลต+ประวัติที่**หน้าสินค้า** · ตอนขายเลือกฉลากตรงอาการ + **แก้วิธีใช้ด่วนเฉพาะบิลได้** · ต่อยอด labelPresets เดิม · หลายภาษา
- **ใบเสร็จ (③)**: **ถามก่อนทุกบิล** "เอาใบเสร็จไหม" (ประหยัดกระดาษ) · ขอใบกำกับภาษี → ออกแบบเต็ม
- **ลำดับพิมพ์**: ฉลากก่อน → ใบเสร็จปิดท้าย (เครื่องความร้อนตัวเดียว)
- → หน้าสินค้าแท็บ "ฉลากยา" ต้องรองรับ **หลายฉลากต่อสินค้า** (mockup อัปเดตแล้ว)

### Flow ขาย POS — แก้ตามที่เจ้าของใช้จริง (18 ก.ค.)
ลำดับจริง: (1) สแกนบาร์โค้ด/QR หรือคีย์ชื่อสินค้า → เพิ่มตะกร้า (วนซ้ำ) → (2) มียาต้องพิมพ์ฉลากไหม? มี→พิมพ์ ไม่มี→ข้าม → (3) มีส่วนลดไหม? มี→ใส่ ไม่มี→ข้าม → (4) มีสมาชิกไหม? กรอกสมาชิก(แต้ม+เตือนแพ้ยา) → (5) รับเงิน → จบบิล
- **จุดใหม่ที่เพิ่ม**: **ส่วนลด** (ตอนรับเงิน) · **ที่อยู่บริษัทลูกค้าขึ้นบิล** (ถ้าปริ้น) → สมาชิกต้องเก็บ ชื่อบริษัท+ที่อยู่+เลขผู้เสียภาษี (ลูกค้าองค์กร) เพื่อออกใบกำกับภาษีเต็ม
- ข.ย./ตัดสต็อก/แต้ม = ระบบทำหลังจบบิลอัตโนมัติ (ไม่ใช่ขั้นที่พนักงานทำ)

### จอขาย POS — เพิ่มแถบปุ่มเครื่องมือ (18 ก.ค. ล้อ hygeia)
- **ฉลาก = ปุ่ม "พิมพ์ฉลาก (F4)"** ให้พนักงานกดเอง (ไม่ใช่ popup เด้งถาม)
- แถบปุ่มขวาสุด: รับเงิน(F3) · พิมพ์ฉลาก(F4) · ใบสั่งยา · เริ่มขายใหม่(F1) · ชุดสินค้า · ลดท้ายบิล(F5) · รับคืนสินค้า · เปิดลิ้นชัก(F7) · เมนู "เพิ่มเติม"

### Flow รับของเข้า — map แล้ว (18 ก.ค.) 2 แบบ
- **แบบ 1 รับจากการซื้อ**: สินค้าใกล้หมด→ใบสั่งซื้อ(หรือสั่งเอง) → ของมา+บิล → กด "รับเข้า" เลือกใบสั่งซื้อ/ผู้ขาย → กรอกแต่ละรายการ(จำนวน·ทุน/หน่วย·เลขล็อต·วันหมดอายุ·เครดิต) → บันทึก=สร้างล็อตใหม่(FIFO)รายสาขา → ลง ข.ย.9 อัตโนมัติ(ยาควบคุม)
- **แบบ 2 รับจากปรับสต็อก**: เจอไม่ตรง/ของแถม/แก้ยอด → กด "ปรับสต็อก" เลือกสินค้า → ระบุ +/− จำนวน+เหตุผล(ใส่ล็อต/วันหมดอายุได้ถ้าเพิ่ม) → บันทึก=ปรับล็อต+เก็บประวัติ(ตรวจสอบได้) · ไม่ต้องมีทุน/ผู้ขาย · ไม่เข้า ข.ย.9
- ทำจากฝั่งจัดการร้าน · เลขล็อต+วันหมดอายุที่กรอกตอนรับ = ตัวที่โผล่ในบัญชี ข.ย. + เตือนใกล้หมดอายุ

### Flow รับของเข้า — รายละเอียดเพิ่ม (18 ก.ค. เจ้าของ)
- **รับจากซื้อ**: กรอกหัวบิล(เลขที่บิล·วันที่ซื้อ·วันที่รับตามเวลาจริง) → ยิง/กรอกบาร์โค้ดสินค้า(จำนวน·ล็อต·วันหมดอายุ·ทุน) วนทุกตัว → บันทึก=สร้างล็อตใหม่+ลง ข.ย.9 → **เด้งถาม "ปริ้น ข.ย.9 เลยไหม?"** ปริ้นเลย/ไม่ปริ้น(ไปปริ้นย้อนหลังที่รายงาน GPP ได้)
- **รับจากปรับสต็อก**: เหมือนกันแต่ **ไม่กรอกหัวบิล/ผู้ขาย · ไม่ลง/ไม่ปริ้น ข.ย.9** · ใช้กับของแถม/ปรับยอด/ของที่ไม่ได้ซื้อมาขายส่ง

### คำตอบเจ้าของ — คำถามสำคัญ (18 ก.ค.)
1. **ฐานสินค้า**: hygeia export ได้ (2 สาขา) → นำเข้า Excel · **สาขาที่ 3 ไม่ได้ใช้ hygeia** → ต้อง "เดินนับสต็อกตั้งต้น" เอง (เริ่มด้วยนับสต็อก)

6. ราคาต่อสาขา: ยังไม่เคาะชัด (เดาว่าเท่ากันทุกสาขาก่อน)
7. **ราคาสมาชิก / สะสมแต้มแลกรางวัล**: มีแผนทำ — สมาชิกได้ราคาพิเศษ หรือสะสมแต้มแลกของรางวัล (เก็บไว้เฟสสมาชิก)
8. **ใบกำกับภาษี = อิสระ** (ออกได้ยืดหยุ่น ไม่บังคับเลขรันนิ่งตายตัว)

### Flow ที่เหลือ — map ครบแล้ว (18 ก.ค.)
- **เปิด-ปิดร้าน**: ใช้ระบบเงินเดิมทั้งหมด · จุดเปลี่ยน: **ยอดขายเงินสดมาจาก POS อัตโนมัติ** (เลิกคีย์มือตอนปิดร้าน) · นับลิ้นชักแยกชนิดเงินเหมือนเดิม → เทียบตรง/ขาด/เกิน · ถ้าขาย offline ต้องซิงค์ก่อนปิดร้าน (ระบบเตือนบิลค้างซิงค์)
- **คืนสินค้า**: ปุ่มรับคืนที่ POS → ค้นบิล → เลือกรายการ/จำนวน → บังคับเหตุผล + audit ชื่อคนทำ → คืนเงิน (สด/โอน/แต้ม) → คืนสต็อกเข้าล็อตเดิม + แก้ ข.ย. + ตัดแต้มที่เคยได้ + ใบรับคืน · มีรายงานการคืน (ใคร/กี่ครั้ง/เหตุผล/มูลค่า)
- **นับสต็อก (มือถือ)**: เริ่มรอบนับ → สแกน+กรอกทีละตัว (หลายคน/หลายวันได้) → เทียบนับจริง vs ระบบ → ยืนยันปรับ (= รับจากปรับสต็อก) · **ใช้เปิดสาขาที่ไม่มี hygeia** (สินค้ามาจาก Excel แล้ว เดินนับใส่จำนวนตั้งต้น)
- **แบ่งยาข้ามสาขา**: A สร้างคำขอ (เห็นว่าสาขาไหนมีเท่าไหร่) → B ได้ push อนุมัติ/ปฏิเสธ → B ส่ง → A กดรับ (ยืนยัน 2 ฝั่ง) → ระบบ**ย้ายยกล็อต** (เลขล็อต/วันหมดอายุ/ทุนติดไปด้วย — ข.ย.+FIFO แม่นทั้ง 2 สาขา) · ทุกคำขอมีประวัติ
- **GPP รายเดือน**: เลือกเดือน+สาขา → ระบบรวบรวม (ข.ย.9←รับเข้า · ข.ย.10/11←บิลขายแยกล็อต · ข.ย.12←ใบสั่ง · อุณหภูมิ←รายวัน) → พรีวิว (ไฮไลต์ข้อมูลขาด) → พิมพ์ทั้งชุด A4 WiFi + กราฟ → เภสัชเซ็นเก็บแฟ้ม · พิมพ์ย้อนหลังได้เสมอ

**สถานะ: map flow ครบ 7 flow แล้ว** → ขั้นต่อไป: เคาะรอบสุดท้าย → สรุปแผนเฟส 1-2-3 ฉบับจริง → เริ่มเฟส 1 (mockup ก่อนโค้ดทุกจอ)
